## 2026-07-11 - US-002
- Implemented keyword-based intent filtering to fix incorrect weather tool invocation
- Files changed:
  - modules/tool_registry.py: Added TOOL_TRIGGER_KEYWORDS dict, get_tools_matching_query(), get_tool_schemas_for_query()
  - modules/commands/llm_command.py: Updated _execute_with_tools() to use filtered tool schemas
- **Learnings for future iterations:**
  - The root cause was that the LLM was being offered all tools regardless of query relevance
  - Solution: Pre-filter tools based on keyword matching before offering to LLM
  - Use word boundaries (\\b) for single-word keywords to avoid false positives (e.g., "hot dog" shouldn't match "hot")
  - Use substring matching for multi-word phrases (e.g., "clear skies")
  - Avoid generic keywords like "hot", "cold", "temp" that can appear in non-domain contexts
  - When no tools match, fall back to direct LLM response (no tools offered)
