## Codebase Patterns
- Database migrations use sequential IDs (1, 2, 3...) not arbitrary numbers
- Migration functions are named _mNNNN_description(cursor) where NNNN matches the function name
- Migrations are appended to MIGRATIONS list as (ID, "description", function)
- Use CREATE TABLE IF NOT EXISTS and CREATE INDEX IF NOT EXISTS for idempotency
- Tests expect migration IDs to be contiguous (range(1, len(MIGRATIONS)+1))
- Use tmp_path fixture for test databases (not :memory:)
