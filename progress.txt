# LLM Tool Calling Integration - Progress Log

## Codebase Patterns
- Database migrations go in modules/db_migrations.py with sequential IDs
- Migrations use helper functions like _add_column() for safe column additions
- Each migration is a function named _mNNNN_description(cursor)
- Migrations are registered in MIGRATIONS list as (version, description, function)
- Never modify existing migrations - always add new ones
- Use _column_exists() to make migrations idempotent
- Test migrations by adding tests in tests/test_db_migrations.py
- Use tmp_path fixture for test databases (not :memory:)
- Typecheck with: .venv/bin/mypy modules/db_migrations.py --config-file=pyproject.toml
- Lint with: .venv/bin/ruff check modules/
